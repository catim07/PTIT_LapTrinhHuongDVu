 Tech Lead Audit Report: Kế Hoạch Nâng Cấp Kép & Vá Lỗi Giao Tiếp Backend-Frontend
Dựa theo Checklist 5 bước mà bạn đã đề ra, tôi đã tiến hành "cross-check" toàn bộ hệ thống từ Database, Backend Controllers, Frontend Services cho đến Redux Slices. Rất nhiều lỗ hổng trong mảng Logistic và Branch-Awareness đã được phát hiện.

Dưới đây là Báo cáo Điểm Mù (Blind Spots) kèm theo Giải pháp vá lỗi (Patch Plan) gửi đến bạn để phê duyệt:

1. Đồng bộ Luồng Dữ liệu (Business Logic Gaps) 🚨
WARNING

Mảng Logistic và Xử lý kho hiện tại đang bị đứt gãy kết nối giữa Controller và Service.

Thiếu hụt (Điểm mù):

Đặt hàng: backend/controllers/orderController.js hàm create hiện chỉ mới tạo record Order vào database, nhưng KHÔNG hề gọi hàm trừ kho inventoryService.deductInventoryForOrder. Dẫn đến việc kho hàng luôn giữ nguyên số lượng dù khách đã đặt.
Hoàn trả: Tương tự, hàm refund trong orderController chỉ đổi trạng thái thành RETURNED/REFUNDED, nhưng KHÔNG có code hoàn kho lại vào Lô hàng (Batch) cũ cũng như hoàn tiền vào Ví (Wallet).
Công nợ Supplier: Các Service supplierService.js đã được viết nhưng hoàn toàn mồ côi (orphaned), không có Route đón request, và Backend API vắng bóng Supplier, PurchaseOrder. Do đó Admin Suppliers Page hoàn toàn không hoạt động thực tế.
Giải pháp vá lỗi (Patch):

Mở file orderController.js và import inventoryService. Gắn hàm chặn vào lúc create đơn hàng để trừ đúng tồn kho từ InventoryBatches.
Gắn logic gọi hàm hoàn kho trong service khi Order bị refund.
Khởi tạo Endpoint Routes (backend/routes/suppliers.js, purchaseOrders.js, v.v.) và kết nối chúng vào ứng dụng.
2. Đồng bộ Chi nhánh (Branch-Awareness Strict Check) 🏢
TIP

Frontend phía User đã hoạt động khá tốt, nhưng Admin Dashboard đang bị lọt dữ liệu tổng.

Thiếu hụt (Điểm mù):

User Side: Hàm đổi Chi nhánh ở Header.tsx có gọi dispatch(clearCart()) ở Redux để xóa UI, nhưng backend cart không bị xóa. Nên nếu người dùng F5 thì cart cũ của chi nhánh trước lại bị load lên bảng.
Admin Side: Trang AdminDashboard.tsx vẫn hiện Data của tất cả các chi nhánh (Tổng Doanh thu toàn quốc), chứ không hề lấy currentBranch.id từ Redux để lọc API.
Giải pháp vá lỗi (Patch):

Cập nhật cartController.js ở Backend cho phép gửi request /clear và nhét nó vào Header.tsx khi User đổi nhánh.
Sửa AdminDashboard.tsx và Dashboard API để cho phép truyền /dashboard?branch_id=HCM01 để Dashboard hiển thị số liệu theo đúng chi nhánh.
3. Rà soát Đa ngôn ngữ (i18n Completeness) 🌍
NOTE

Ngôn ngữ chưa đi sâu vào từng ngóc ngách của Data động.

Thiếu hụt (Điểm mù):

Các badge hiển thị Status như "PENDING", "RETURNED", "DELIVERED" ở các trang quản lý đơn hàng vẫn đang hard-code tag tiếng Anh.
Giải pháp vá lỗi (Patch):

Khai báo thêm danh mục orderStatuses vào vi.json và en.json.
Gắn t('status.PENDING') vào tất cả các Badge để chúng nhảy ngôn ngữ chính xác.
4. Trải nghiệm & Giao diện (UI/UX Sync) 🎨
Thiếu hụt (Điểm mù): Trang Admin Dashboard chưa được tận dụng triệt để mảng hàng hóa (Low Stock Warning).
Giải pháp vá lỗi (Patch): Tôi sẽ tạo một hàm API ở inventoryService để trả về bảng cảnh báo sản phẩm low_stock (Dưới 10 chai/gói) và expiring_soon rồi gắn nó thành bảng Widget cảnh báo ngay tại trang AdminDashboard.tsx.
5. Data Service Của Frontend 🔌
IMPORTANT

Backend có Service nhưng Frontend chưa có Axios fetch!

Thiếu hụt (Điểm mù): dataService.ts trên FE trải dài hơn 600 dòng nhưng hoàn toàn vắng bóng các hàm API Call cho: Suppliers, PurchaseOrders, InventoryBatches, StockTakes, InternalRequisitions. Đứt gãy toàn mạch trang Admin Management.
Giải pháp vá lỗi (Patch):
Bổ sung toàn bộ CRUD RESTful endpoints (khoảng 30 hàm) cho các Module Enterprise này vào thẳng file dataService.ts phía client.
