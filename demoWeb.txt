BÁO CÁO KỸ THUẬT & HƯỚNG DẪN DEMO HỆ THỐNG LOTTE MART (MERN STACK)
--------------------------------------------------------------------------------

I. TỔNG QUAN HỆ THỐNG (SYSTEM OVERVIEW)
Hệ thống Lotte Mart Ecommerce được xây dựng dựa trên kiến trúc MERN Stack (MongoDB, Express, React, Node.js).
Giai đoạn nâng cấp gần nhất đã chuyển đổi toàn bộ dự án từ việc sử dụng dữ liệu giả (mockData.json) sang môi trường tích hợp API thực tế, truy xuất trực tiếp từ cơ sở dữ liệu MongoDB. Mọi tính năng hiển thị từ trang chủ, sự kiện, đến chi tiết sản phẩm đều hoạt động 100% qua luồng API chuẩn hóa.

II. KIẾN TRÚC HỆ THỐNG (ARCHITECTURE)
1. Frontend: React.js + TypeScript + Tailwind CSS (Vite)
2. Backend: Node.js + Express.js
3. Database: MongoDB (Mongoose ODM)
4. Giao tiếp: RESTful API qua Axios (mô hình Request/Response)

---

III. LUỒNG DỮ LIỆU CHI TIẾT (DATA FLOW)

1. TRANG CHỦ (Home Page - Banners & Products Sections)
[UI] Người dùng truy cập trang chủ / 
  → [FRONTEND] Home.tsx kích hoạt useEffect()
  → [SERVICE] httpClient.get('/banners') và httpClient.get('/products?featured=true...')
  → [API] Gửi HTTP GET request tới Backend (Port 3001)
  → [BACKEND] Router nhận request tại banners.js / products.js
  → [CONTROLLER] bannerController.js / productController.js xử lý logic, query DB
  → [DATABASE] MongoDB tìm kiếm trong collections `banners` & `products` 
  → [RESPONSE] Trả về JSON chứa dánh sách Banner và Sản phẩm
  → [FRONTEND] Home.tsx nhận data, lưu vào React State (setBanners, setFeaturedProducts)
  → [UI] Render giao diện slider và lưới sản phẩm.

2. SỰ KIỆN NỔI BẬT (Featured Events)
[UI] Điều hướng vào /featured-events 
  → [FRONTEND] FeaturedEvents.tsx gọi eventService.getFeaturedEvents()
  → [API] GET /api/events?featured=true&limit=6
  → [BACKEND] eventController.js xử lý tìm kiếm bài viết is_published=true và is_featured=true
  → [DATABASE] Truy vấn collection `event_posts`
  → [UI] Cập nhật giao diện với danh sách sự kiện hấp dẫn mới nhất.

3. CHI TIẾT SẢN PHẨM (Product Detail)
[UI] Click vào một sản phẩm cụ thể 
  → [FRONTEND] ProductDetail.tsx lấy `id` từ URL (useParams)
  → [API] GET /api/products/:id
  → [BACKEND] productController.detail kết hợp aggregation pipeline (kết hợp BranchProduct để lấy giá theo chi nhánh)
  → [DATABASE] Truy vấn collection `products` + join `branchproducts`
  → [UI] Render thông tin chi tiết, giá tiền, rating.

IV. BẢNG MAPPING API (API MAPPING TABLE)

| TÍNH NĂNG            | METHOD | API ENDPOINT                                | CHỨC NĂNG DỮ LIỆU                             |
|----------------------|--------|---------------------------------------------|-----------------------------------------------|
| Banners Trang Chủ    | GET    | /api/banners                                | Lấy 3 banners đang active (is_active: true)   |
| Sản phẩm nổi bật     | GET    | /api/products?featured=true&branchId=HCM01  | Lọc sản phẩm có cờ is_featured = true         |
| Sản phẩm mới         | GET    | /api/products?sort=newest&branchId=HCM01    | Lọc sản phẩm sắp xếp theo thời gian mới nhất  |
| Sản phẩm bán chạy    | GET    | /api/products?sort=best&branchId=HCM01      | Lọc sản phẩm sắp xếp theo lượt bán khủng nhất |
| Sự kiện nổi bật      | GET    | /api/events?featured=true&limit=6           | Lấy top 6 bài viết sự kiện nổi bật            |
| Chi tiết sự kiện     | GET    | /api/events/:slug                           | Lấy toàn bộ thông tin bài event theo slug     |
| Chi tiết sản phẩm    | GET    | /api/products/:id                           | Lấy thông tin gốc + tồn kho/giá tại chi nhánh |

VI. CẤU TRÚC DATABASE (DATABASE STRUCTURE)
Tất cả dữ liệu được tổ chức dưới dạng NoSQL Documents trong MongoDB:
1. `products`: Gồm thông tin chuẩn của hàng hóa (name, category, brand, images).
2. `branchproducts`: Lưu giá, tồn kho (stock) của một sản phẩm tại một siêu thị cụ thể.
3. `banners`: Lưu trữ ảnh bìa quảng cáo, vị trí hiển thị (home, promo).
4. `event_posts`: Cấu trúc bài báo sự kiện (tiêu đề, slug, nội dung thumbnail, author, views, likes).

VII. CÁC VẤN ĐỀ ĐÃ KHẮC PHỤC (ISSUES & FIXES)

1. Cắt đứt hoàn toàn "mockData.json": Toàn bộ Frontend đã được quét (grep) và loại bỏ hoàn toàn việc import từ file giả lập nội bộ. Dữ liệu hiện nay 100% trích lập theo đường chuẩn API.
2. Fix API Endpoint Crash / Lỗi mạng: Xử lý file `dataService` và `endpoints.ts` để đồng bộ đúng cổng `:3001` sang API máy chủ. Thiết lập HTTP Client xử lý tự động Catch() lỗi.
3. Xử lý "Bể UI" Trang chủ / Trang sự kiện thiếu data: 
   - Thay vì lỗi trắng trang do không có backend trả dữ liệu, hệ thống tự động đổ Skeleton Loaders bóng bẩy hoặc thông báo "Không có sản phẩm" rất tinh tế.
   - Thêm Script Seed data tự động (seedEvents.js, seedBanners.js) bắn thẳng data chuẩn vào DB phòng trường hợp cơ sở dữ liệu trống.
4. Tách biệt Data Model Events: Tạo mới Mongoose Schema `EventPost` tương tự format chuẩn, khắc phục lỗi không mapping được tham số.
